# 버그 수정 및 리팩토링 실행 계획서

**작성일**: 2026-05-10  
**대상 문서**: `0508.pdf` 10건 + `0510.pdf` 17건 = 총 27건  
**목표**: 3D 화면, 2D 도면, PDF, DXF, 패널 목록이 같은 치수/형상 기준을 사용하도록 수정한다.  
**상태**: 구현 전 실행 계획

---

## 0. 결론

이 작업은 단순 버그 묶음 수정으로 처리하면 안 된다.

현재 오류 대부분은 개별 UI 문제가 아니라 다음 구조 문제에서 나온다.

1. 도어 H/W/Y 계산이 `DoorModule`, `CADDimensions2D`, `CleanCAD2D`, `calculatePanelDetails`, `usePDFExport`에 흩어져 있다.
2. 선반 들이기, 신발장 깊이, 찬넬 폭, EP 갭 같은 모듈 규칙이 `moduleId.includes(...)` 분기로 반복된다.
3. 2D 화면과 PDF/DXF가 항상 같은 데이터를 쓰는 구조가 아니다.
4. 새 모듈이 추가될 때 렌더, 치수, export 중 하나가 누락될 가능성이 높다.

따라서 작업 순서는 반드시 다음과 같이 간다.

```text
Phase -1  기준 잠금
Phase 0   공통 계산기 관찰 모드 추가
Phase 1   독립 저위험 수정
Phase 2   단일 기능 수정
Phase 3   공통 계산기로 교체
Phase 4   데이터 의미 변경/마이그레이션
```

## 1. 절대 원칙

### 1.1 큰 리팩토링 금지

한 번에 `DoorModule`, `CADDimensions2D`, `calculatePanelDetails`, PDF export를 모두 바꾸지 않는다.  
먼저 현재 정상 출력 기준을 고정하고, 새 계산기는 기존 결과와 비교하는 관찰 모드로 붙인다.

### 1.2 기준 출력 없이는 수정 금지

다음 항목은 작업 전 기준값을 저장해야 한다.

- 3D 화면
- 2D 정면뷰
- 2D 평면뷰
- 2D 좌측뷰
- 도어 도면
- PDF 출력
- DXF 출력
- 패널 목록/컷리스트

### 1.3 측면도 정책

발주/export 측면도는 **좌측뷰만** 사용한다.  
우측뷰 UI/템플릿/export 경로가 남아 있으면 제거하거나 비활성화한다.

### 1.4 커밋 단위

커밋은 다음 단위로 끊는다.

- 기준 테스트/스냅샷 추가
- 공통 계산기 추가
- 호출부 교체 1개 축
- 버그 수정 1건 또는 강하게 연관된 묶음

## 4. Phase 0 — 공통 계산기 추가

**중요**: Phase 0에서는 기존 렌더 결과를 바로 바꾸지 않는다.  
새 계산기는 기존 값과 비교하는 관찰 모드로 먼저 넣는다.

### 4.1 도어 계산기

신규 파일:

```text
src/editor/shared/utils/doorGeometryCalculator.ts
```

담당 범위:

- 도어 W
- 도어 H
- 도어 Y 위치
- 도어 상단/하단 절대 좌표
- 1짝/2짝 판정
- 상부장/하부장/키큰장/팬트리/멍장/인덕션/상판내림
- 18T/18.5T/30T 상판 보정
- 띄움장/걸레받이/바닥마감 보정

초기 API:

```ts
resolveDoorGeometry(input): {
  leafCount: 1 | 2
  leaves: Array<{
    widthMm: number
    heightMm: number
    bottomMm: number
    topMm: number
    centerYMm: number
  }>
  warnings: string[]
}
```

기존 계산과 비교할 호출부:

- `DoorModule.tsx`
- `CADDimensions2D.tsx`
- `CleanCAD2D.tsx`
- `calculatePanelDetails.ts`
- `usePDFExport.ts`

### 4.2 선반 들이기 계산기

신규 파일:

```text
src/editor/shared/utils/shelfInsetCalculator.ts
```

API:

```ts
resolveShelfFrontInsetMm({
  moduleId,
  depthMm,
  sectionType,
  cabinetCategory
}): number
```

주의:

- PDF에는 “깊이 크면 72, 작으면 20”만 있다.
- 임계값은 사용자 확정 전까지 상수로 박지 않는다.
- 임계값 확정 전 구현 시 `SHELF_INSET_DEPTH_THRESHOLD_MM` 이름으로 한 곳에 둔다.

호출부:

- `ShelfRenderer.tsx`
- `SectionsRenderer.tsx`
- `BoxModule.tsx`
- `SingleType2.tsx`
- `DualType2.tsx`
- `UpperCabinet.tsx`
- `LowerCabinet.tsx`
- `CustomizableBoxModule.tsx`
- `calculatePanelDetails.ts`

### 4.3 치수 계산기

기존 파일:

```text
src/editor/shared/utils/furnitureDimensionCalculator.ts
```

보강 범위:

- 폭 반올림 기준 통일
- `1100`/`1101` 1mm 오차 기준 정리
- 측면 H 20mm 오차 기준 정리
- 상판/바닥/프레임/EP/도어 포함 여부 명시

출력 형태:

```ts
resolveDrawingDimensions(input): {
  front: DimensionSegment[]
  top: DimensionSegment[]
  left: DimensionSegment[]
  door: DimensionSegment[]
}
```

### 4.4 모듈 판별 어댑터

신규 파일:

```text
src/editor/shared/utils/moduleClassification.ts
```

목적:

- `moduleId.includes(...)` 분기를 한 곳으로 모은다.
- 새 모듈 추가 시 렌더/export/치수 누락을 줄인다.

예시:

```ts
classifyModule(moduleId): {
  family: 'upper' | 'lower' | 'full' | 'entryway' | 'pantry' | 'channel' | 'dummy' | 'induction'
  isShoeCabinet: boolean
  isDrawerCabinet: boolean
  isDoorLift: boolean
  isTopDown: boolean
  isChannel: boolean
}
```
